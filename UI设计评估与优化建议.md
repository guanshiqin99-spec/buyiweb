# 布依族词典 Web 版 UI 设计评估与优化建议

更新时间：2026-07-10

本文档基于当前 Vue 前端实现，重点评估整体 UI 观感、液体玻璃材质的质感问题，以及后续可执行的优化方向。相关实现主要集中在 [src/assets/styles/liquid-glass.css](src/assets/styles/liquid-glass.css)、[src/assets/styles/variables.css](src/assets/styles/variables.css)、[src/components/common/PageShell.vue](src/components/common/PageShell.vue)、[src/components/layout/AppHeader.vue](src/components/layout/AppHeader.vue)、[src/views/Home.vue](src/views/Home.vue)、[src/views/Dictionary.vue](src/views/Dictionary.vue) 等文件。

## 一、先给结论

当前 UI 的方向是对的：主题统一、叙事性强、民族文化氛围明确，页面也有完整的视觉系统，而不是零散拼接的样式。问题主要不在“有没有风格”，而在“风格太满、太平均、太用力”，尤其是液体玻璃被过度使用后，视觉语言开始失去层级，出现一种“透明、发亮、会浮起，但不够高级”的感觉。

你提到的“液体玻璃有一点点廉价”，这个判断是成立的。主要原因不是玻璃本身，而是当前实现把玻璃效果、发光、粒子、背景图、渐变和动效叠得太密，导致材质像装饰插件，而不是产品的主视觉秩序。

## 二、当前做得不错的地方

1. 主题方向统一，靛蓝 + 暖橙 + 米白的组合有识别度，也契合少数民族文化叙事。
2. 主页、词典页、个人中心等页面都有明确的主次层级，不是纯信息表单风格。
3. 页面里已经有较完整的响应式、无障碍和减少动效处理，基础质量不差。
4. 设计系统已经有令牌化倾向，后续是可以通过调令牌把质感拉升的，不需要推倒重做。

## 三、为什么会显得“廉价”

### 1. 玻璃效果覆盖面太大

当前不少页面和组件都在用 [liquid-glass](src/assets/styles/liquid-glass.css) 和 [glow-card](src/assets/styles/liquid-glass.css) 一类效果。结果是：主卡片、次级卡片、统计卡、菜单列表、播放器、弹窗都像同一种材质，只是大小不同。

这种做法的问题是，视觉上没有“重要 vs 次要”的分层。用户会感到每个块都在抢戏，反而没有真正的焦点。

### 2. 高光、边框、阴影都偏满

液体玻璃当前有明显的高光、菲涅尔边框、多重阴影、渐变厚度层、噪点层和额外 glow 层。单独看每个细节都合理，但叠起来之后，表面会显得“很努力地在像高级材质”，反而缺少克制。

### 3. 页面层级和背景层级都偏复杂

像 [src/components/common/PageShell.vue](src/components/common/PageShell.vue) 和 [src/views/Home.vue](src/views/Home.vue) 这类页面，背景图、遮罩、粒子、视差、玻璃卡片同时存在。信息层级本来已经够丰富，再加大量材质装饰，就会让视觉重心发散。

### 4. 动效太像“默认高级感模板”

当前很多地方都在做 hover 浮起、呼吸、渐入、粒子漂浮、高光移动。单个动效都不差，但组合起来容易形成一种常见的“AI 生成感”或“设计模板感”，缺少更明确的品牌节奏。

### 5. 一些局部细节不够统一

比如搜索、导航、设置、播放器的交互态虽然都可用，但按钮高度、边框强度、圆角、文字权重并不完全统一。用户不一定能说出是哪儿不对，但会感到“像同一个系统，又不是完全同一个系统”。

## 四、除了液体玻璃，UI 还有哪些问题

### 1. 信息层级过多，但主次不够硬

当前页面普遍有标题、标签、说明、按钮、卡片、背景图、徽章、图表等多层信息，但真正能一眼抓住的“第一重点”不够明确。结果是内容不散，但不够聚焦。

### 2. 页面很完整，但缺少页面性格差异

首页、词典页、个人中心、设置页都在使用相近的视觉语法，导致产品像一个统一模板在不断复用，而不是不同功能页有不同气质。好的产品会让人一眼分出“展示页、工作页、管理页、收尾页”的差异。

### 3. 功能性页面和氛围性页面的边界不够清楚

词典、搜索、设置、登录这类页面，本质上是工具；首页、文化、民歌这类页面，本质上是叙事。现在这两类页面的视觉表达偏接近，导致工具页也有一点“表演感”，叙事页又不够沉浸。

### 4. 交互反馈略偏统一，缺少层级反馈

很多按钮、卡片、列表项的 hover、active、focus 反馈很接近，视觉反馈虽然一致，但不够区分“主动作”和“次动作”。比如主按钮、次按钮、列表项、标签都应该有不同程度的反馈力度。

### 5. 空状态、弱状态、默认状态之间的区分不够强

登录前、暂无数据、加载中、可操作、已选中这几种状态，现在有被照顾到，但视觉上还没有被严格区分。用户在使用时会觉得“界面是对的”，但操作路径的引导性还可以更清晰。

### 6. 页面节奏偏平均，缺少真正的停顿点

当前很多页面从头到尾都在提供信息和视觉刺激，少有“空一拍”的区域。优秀的 UI 会主动留出呼吸区，让用户在一两个关键位置停下来理解内容，而不是持续被填满。

### 7. 文化表达有，但符号化程度偏浅

现在已经有靛蓝、纹样、民歌、蜡染等元素，但它们更多是“装饰符号”，还没有进一步转化为排版秩序、网格结构、插图语言和页面节奏。换句话说，文化元素已经进来了，但还没有完全融进系统。

## 五、具体哪些小细节可以再打磨

### 1. 卡片密度和圆角节奏不统一

有些卡片偏圆、偏软，有些偏硬、偏紧。建议把卡片分成明确层级，而不是都用同一种玻璃半径和同一种阴影强度。

### 2. 文字层级还可以更克制

部分标题和按钮强调得比较多，导致页面里“每一层都重要”。建议减少同时发声的元素，让每屏只保留一个真正的视觉主角。

### 3. 颜色的“通透感”可以再收一点

现在玻璃和背景色都偏亮，尤其浅色模式下，玻璃面板有点像“白色半透明卡片”。如果把底色压暗一点、把高光收窄一点，质感会更像真实材质而不是特效。

### 4. 空状态和普通内容状态的差异还不够

当前很多空状态、统计卡、内容卡都共享相同的材质语言，缺少“空的时候更轻，内容的时候更实”的区别。这个细节一做，产品会立刻更成熟。

### 5. 图片和玻璃的关系还可以更讲究

像首页和文化页这种图片很多的页面，如果所有图片外层都再加玻璃和高光，容易变花。更好的做法是让图片本身承担氛围，外层材质只做少量收边。

## 六、液体玻璃的改进方法和策略

### 策略一：把玻璃从“全局效果”改成“重点效果”

这是最重要的一条。

建议只保留 3 个层级：

1. 重点层：用于首页 Hero、搜索弹窗、播放器等高关注区域。
2. 内容层：用于详情卡、表单卡、统计卡，但弱化高光和阴影。
3. 底层：用于普通列表、菜单、设置项，改成更克制的半透明或轻浮雕风格。

这样一来，玻璃才会有“身份”，不会每个组件都像在同一个展柜里发光。

### 策略二：降低“光学特效感”，增加“材料感”

目前的玻璃更像光学特效，不像真实材质。建议做这些调整：

1. 降低 blur 和 saturation，避免表面太“奶”。
2. 减少高光面积，让高光更集中、更窄。
3. 减弱多层 inset 叠加，保留一到两层最有效的边缘厚度。
4. 降低噪点纹理存在感，避免面板显脏。
5. 将 hover 提升幅度收小，不要所有卡片都明显上浮。

### 策略三：让背景更克制，给前景留呼吸

建议减少同时存在的背景元素数量。比如粒子、重渐变、强遮罩、强视差不要在所有页面都一起用。背景越安静，玻璃才越显得高级。

更推荐的组合是：

1. Hero 页：背景图 + 大面积 scrim + 一处玻璃。
2. 内容页：更平静的纯色/弱纹理背景 + 少量玻璃卡。
3. 工具页：以清晰可读为先，玻璃只做点缀。

### 策略四：建立明确的材质语义

不要把“玻璃”当成一个装饰词，而要把它变成产品语义。

建议定义几种可复用的材质语义：

1. Hero Glass：更厚、更亮、更有景深。
2. Content Frost：轻微模糊，强调内容可读性。
3. Quiet Panel：几乎不发光，只保留边界和轻阴影。
4. Action Pill：用于按钮、筛选、播放器控制，强调可点击性。

这样后续新页面即使继续用玻璃，也不会越来越乱。

### 策略五：把动效从“处处有”改成“少数地方有”

当前的呼吸、浮起、渐入和光斑很多，建议只保留真正增强体验的动效。

优先保留：

1. 首页 Hero 的入场。
2. 搜索和播放器等高频交互的反馈。
3. 页面切换时轻微的内容出现。

建议减少：

1. 所有卡片统一 hover 浮起。
2. 所有玻璃都跟鼠标光斑。
3. 所有标题都呼吸发亮。

## 七、整体 UI 的优化方向

### 方向 1：从“炫技风”转向“馆藏感”

这个项目很适合做成“数字文化馆”而不是“炫光科技站”。如果把重点放在文化叙事、内容秩序和材质克制上，反而更符合主题，也更容易显高级。

### 方向 2：从“统一特效”转向“统一系统 + 分层表达”

统一不等于所有地方一样。真正高级的系统，是基础规则统一，但不同页面有不同的表达权重。

建议首页更有展陈感，词典页更强调功能性，个人中心更强调信息整合，设置页更偏工具化。不要让所有页面都像一个风格滤镜下的复制品。

### 方向 3：从“发光”转向“留白 + 质感”

现在的视觉重心有点偏“发光”。如果增加一点留白、收缩一点阴影、减少一点高光，整个界面会更稳。

### 方向 4：从“装饰先行”转向“信息先行”

词典产品的核心是查找、阅读和使用，不是观赏特效。建议确保任何装饰都不能压过搜索、结果、解释、学习等主任务。

## 八、如果要落地，建议的改造顺序

1. 先改 [src/assets/styles/liquid-glass.css](src/assets/styles/liquid-glass.css)，降低玻璃强度，建立三档材质层级。
2. 再收 [src/components/common/PageShell.vue](src/components/common/PageShell.vue) 和 [src/views/Home.vue](src/views/Home.vue) 的背景复杂度，让首页主视觉更稳。
3. 统一 [src/components/layout/AppHeader.vue](src/components/layout/AppHeader.vue)、搜索、播放器、设置页的按钮和圆角语言。
4. 之后再细修各页面卡片的透明度、阴影和 hover 幅度。

## 九、可以直接执行的修改原则

1. 每一屏只保留一个“最亮”的视觉重点。
2. 卡片不要都像玻璃，次要内容宁可更平。
3. 高光面积越小，质感越容易高级。
4. 玻璃效果越少，越能显出它的重要性。
5. 动效要服务层级，不要服务炫技。

## 十、最终判断

当前 UI 的骨架是合格的，甚至已经有自己的审美方向了。真正需要修的，不是“换一个完全不同的风格”，而是把现有风格从“特效感”调到“材质感”，从“每处都在发光”调到“少数地方有重量”。

如果按这个方向收一轮，液体玻璃的廉价感会明显下降，整体也会更像一个成熟的文化类产品，而不是一个效果展示页。
