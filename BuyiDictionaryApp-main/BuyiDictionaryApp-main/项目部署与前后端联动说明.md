# 布依语词典项目部署与前后端联动说明

这份文档是给第一次接手这个项目的人看的。

目标只有两个：

1. 让你把项目跑起来
2. 让你看明白前端、后端、数据库到底怎么连在一起

如果你不想看原理，直接先看"**一键理解**"和"**本地跑通**"两节。

---

## 1. 一键理解

这个项目其实分成 3 块：

1. **小程序前端**
   放在项目根目录，像 `pages/`、`components/`、`utils/`、`app.js` 这些都是它的代码。

2. **后台管理前端**
   放在 [admin-web](F:/BuyiDictionaryApp-main/BuyiDictionaryApp-main/admin-web) 目录。
   这是一个纯静态前端页面，给运营人员管理词条、短语、谚语、民歌、媒体资源用。

3. **后端**
   放在 [backend](F:/BuyiDictionaryApp-main/BuyiDictionaryApp-main/backend) 目录。
   它负责：
   - 连接数据库
   - 提供接口
   - 处理登录
   - 提供后台管理接口
   - 提供小程序接口
   - 把后台前端静态页面挂出来

你可以把它理解成下面这样：

```text
小程序前端  --->  请求后端接口  --->  读写数据库
后台前端    --->  请求后端接口  --->  读写数据库
```

也就是说：

- **真正的数据都应该在数据库里**
- 小程序和后台都只是"显示和操作数据的界面"
- 后端是中间层，负责把界面和数据库连接起来

## 2. 这个项目现在是怎么联动的

### 2.1 后端提供两类接口

后端启动后，会提供两套主要接口：

- 小程序接口：`/api/miniapp/*`
- 后台接口：`/api/admin/*`

例如：

- 小程序查词：`/api/miniapp/dictionary`
- 小程序搜索：`/api/miniapp/search`
- 小程序首页数据：`/api/miniapp/home`
- 后台登录：`/api/admin/auth/login`
- 后台词条管理：`/api/admin/dictionary`
- 后台民歌管理：`/api/admin/songs`

### 2.2 后台管理前端怎么连后端

后台前端默认把接口地址写成：

- `/api`

这意味着：

- 如果你访问后台页面地址是 `http://你的域名/admin-web/`
- 那么它请求接口时会自动打到 `http://你的域名/api/...`

这就是为什么后台前端和后端最好部署在**同一个域名**下。

后台页面本身也是由后端直接挂出来的：

- 后台地址：`/admin-web/`

所以最简单的部署方式就是：

- 一个后端服务
- 同时提供 `/api` 接口
- 同时提供 `/admin-web` 页面

### 2.3 小程序前端怎么连后端

小程序通过 [utils/api.js](F:/BuyiDictionaryApp-main/BuyiDictionaryApp-main/utils/api.js) 发请求。

它真正使用的接口地址来自：

- [app.js](F:/BuyiDictionaryApp-main/BuyiDictionaryApp-main/app.js)
- [utils/runtime-config.js](F:/BuyiDictionaryApp-main/BuyiDictionaryApp-main/utils/runtime-config.js)
- 设置页 [pages/setting/index.js](F:/BuyiDictionaryApp-main/BuyiDictionaryApp-main/pages/setting/index.js)

默认逻辑是：

- 开发环境接口：`http://127.0.0.1:3000/api`
- 生产环境接口：`https://api.buyi.example.com/api`

小程序运行时会根据环境决定用哪个地址。

你也可以在小程序"设置"页里手动改开发地址和生产地址。

### 2.4 数据库在里面扮演什么角色

数据库保存的是这些核心数据：

- 词条
- 短语
- 谚语
- 民歌
- 媒体资源
- 用户
- 收藏
- 学习记录
- 设置

现在首页轮播、搜索建议、词条列表、短语、谚语、民歌这些内容，都是通过后端接口从数据库里拿，不应该再靠前端写死数组。

重点是 [backend/.env](F:/BuyiDictionaryApp-main/BuyiDictionaryApp-main/backend/.env)。

生产环境必须改这些值：

```env
NODE_ENV=production

DB_SYNCHRONIZE=false

WECHAT_APP_ID=你的小程序AppID
WECHAT_APP_SECRET=你的小程序AppSecret
WECHAT_MOCK_MODE=false

MEDIA_DRIVER=cos
MEDIA_PUBLIC_BASE_URL=https://你的媒体访问域名
COS_SECRET_ID=你的腾讯云密钥ID
COS_SECRET_KEY=你的腾讯云密钥KEY
COS_BUCKET=你的桶名
COS_REGION=你的地域
COS_PUBLIC_BASE_URL=https://你的COS公开地址

SEED_ON_BOOT=false
DEFAULT_ADMIN_USERNAME=
DEFAULT_ADMIN_PASSWORD=
```

### 这几个生产原则一定记住

1. **生产不要再用 sqljs**
   本地可以，正式环境不要。

2. **生产要改成 MySQL**
   因为数据更稳定，也便于备份。

3. **生产不要开 DB_SYNCHRONIZE**
   否则风险很大。

4. **生产不要开 SEED_ON_BOOT**
   否则每次启动都会尝试插入默认数据。

5. **生产小程序接口必须是 HTTPS**
   微信正式版不接受 HTTP。

## 5.7 配置 Nginx

推荐让 Nginx 反向代理到 Node 服务。

最简单的思路是：

- 用户访问 `https://你的域名/admin-web/`，打开后台页面
- 后台页面请求 `https://你的域名/api/...`，由 Nginx 转给 Node
- 小程序也请求 `https://你的域名/api/...`

示例配置：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /path/to/fullchain.pem;
    ssl_certificate_key /path/to/privkey.pem;

    client_max_body_size 20m;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

这个配置的意思很简单：

- 浏览器访问你的域名
- Nginx 把请求转给 `127.0.0.1:3000`
- 后端自己负责返回 `/api`、`/admin-web`、`/uploads`

## 6. 后台前端怎么部署

这个项目的后台前端不需要你单独打包到 Nginx 静态目录。

因为后端已经在 [backend/src/app.module.ts](F:/BuyiDictionaryApp-main/BuyiDictionaryApp-main/backend/src/app.module.ts) 里把这个目录直接挂出来了：

- [admin-web](F:/BuyiDictionaryApp-main/BuyiDictionaryApp-main/admin-web)

也就是说，只要后端服务启动成功，你访问：

- `https://你的域名/admin-web/`

就能打开后台。

### 这意味着什么

意味着后台前端和后端是"绑在一起部署"的。

优点：

- 部署简单
- 不容易跨域
- 接口地址默认就是 `/api`

缺点：

- 你每次改后台页面，最好和后端一起部署

## 7. 小程序前端怎么部署

小程序不是传统网页前端，它不是扔到服务器就完事了。

它的"部署"流程是：

1. 在微信开发者工具里打开项目
2. 配置小程序后台合法域名
3. 把生产接口地址改成你的正式 HTTPS 地址
4. 上传代码
5. 提交审核
6. 发布

### 7.1 小程序后台要配置什么

你要在微信公众平台里配置合法 request 域名，例如：

```text
https://api.your-domain.com
```

或者如果你是同域：

```text
https://your-domain.com
```

关键是：

- 必须 HTTPS
- 必须和你小程序里实际请求的接口域名一致

### 7.2 小程序生产地址在哪里改

默认配置在：

- [app.js](F:/BuyiDictionaryApp-main/BuyiDictionaryApp-main/app.js)
- [utils/runtime-config.js](F:/BuyiDictionaryApp-main/BuyiDictionaryApp-main/utils/runtime-config.js)

当前项目代码的实际情况是：

- `utils/runtime-config.js` 里开发和生产默认地址都还是 `http://47.114.114.135/api`
- `app.js` 里的 `https://api.buyi.example.com/api` 更像是早期占位值，不是当前真实生效的上线地址

所以如果你继续使用阿里云服务器，上线前必须把"生产环境接口地址"改成你自己的 HTTPS 域名，例如：

```text
https://api.your-domain.com/api
```

不能继续保留 HTTP IP 地址，否则真机预览、体验版、正式版都会有被微信拦截的风险。

另外，项目本身也支持在小程序设置页动态修改接口地址。

### 7.3 继续使用阿里云服务器的可上线方案

如果你准备继续把后端放在阿里云服务器上，推荐按下面这套结构来做。

#### 方案目标

- 后端服务继续跑在阿里云 ECS 上
- Node 服务只监听内网或本机端口，例如 `127.0.0.1:3000`
- 对外只暴露 HTTPS 域名，例如 `https://api.your-domain.com`
- 微信小程序只请求这个 HTTPS 域名

#### 推荐域名结构

最推荐的是前后端分域：

```text
API 域名: https://api.your-domain.com
后台域名: https://admin.your-domain.com
```

如果你想少配一个域名，也可以同域：

```text
统一域名: https://your-domain.com
后台地址: https://your-domain.com/admin-web/
接口地址: https://your-domain.com/api
```

对小程序来说，最关键的是"实际请求接口的那个域名"必须是 HTTPS，并且已经配置到微信公众平台的合法 request 域名里。

#### 阿里云服务器上的部署结构

推荐这样放：

```text
阿里云 ECS
  Nginx : 80 / 443
  Node/NestJS : 127.0.0.1:3000
  MySQL : 127.0.0.1:3306 或独立 RDS
```

访问路径建议这样走：

```text
微信小程序 -> https://api.your-domain.com/api/* -> Nginx -> http://127.0.0.1:3000/api/*
浏览器后台 -> https://admin.your-domain.com/admin-web/ -> Nginx -> http://127.0.0.1:3000/admin-web/
```

#### 必须做的 6 件事

1. 购买并解析域名

   至少准备一个域名，把 `api.your-domain.com` 解析到阿里云服务器公网 IP。

2. 申请 HTTPS 证书

   可以用阿里云证书服务，也可以用 Let\'s Encrypt。证书要绑定你真正给小程序使用的域名。

3. 配置 Nginx 反向代理

   让外部所有 HTTPS 请求都先到 Nginx，再由 Nginx 转发到 `127.0.0.1:3000`。

4. 修改后端生产环境变量

   至少把这些值改对：

   ```env
   NODE_ENV=production
   APP_PUBLIC_BASE_URL=https://api.your-domain.com
   CORS_ORIGIN=https://admin.your-domain.com,https://servicewechat.com
   WECHAT_APP_ID=你的小程序AppID
   WECHAT_APP_SECRET=你的小程序AppSecret
   WECHAT_MOCK_MODE=false
   MEDIA_DRIVER=cos
   ```

   如果你使用同域方案，则可改成：

   ```env
   APP_PUBLIC_BASE_URL=https://your-domain.com
   CORS_ORIGIN=https://your-domain.com,https://servicewechat.com
   ```

5. 修改小程序生产接口地址

   把生产环境地址改成：

   ```text
   https://api.your-domain.com/api
   ```

   如果是同域方案，则改成：

   ```text
   https://your-domain.com/api
   ```

6. 在微信公众平台配置合法域名

   登录小程序后台，在"开发管理 -> 开发设置 -> 服务器域名"里添加：

   ```text
   https://api.your-domain.com
   ```

   或：

   ```text
   https://your-domain.com
   ```

#### Nginx 推荐配置

如果你准备前后端分域，可以参考下面的配置。

```nginx
server {
    listen 80;
    server_name api.your-domain.com admin.your-domain.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name api.your-domain.com;

    ssl_certificate /etc/nginx/ssl/api.fullchain.pem;
    ssl_certificate_key /etc/nginx/ssl/api.key.pem;

    client_max_body_size 20m;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

server {
    listen 443 ssl http2;
    server_name admin.your-domain.com;

    ssl_certificate /etc/nginx/ssl/admin.fullchain.pem;
    ssl_certificate_key /etc/nginx/ssl/admin.key.pem;

    client_max_body_size 20m;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

如果你只想用一个域名，前面的 5.7 小节配置就够用，只要把 `your-domain.com` 换成你的真实域名，并确保小程序请求的是 `https://your-domain.com/api`。

#### 微信侧的上线判断标准

只要下面 4 条同时满足，手机预览版、体验版、正式版就都能访问阿里云后端：

1. 小程序请求的是 HTTPS 域名，不是 HTTP IP
2. 这个域名已经正确解析到阿里云服务器
3. Nginx 证书有效，浏览器直接访问接口域名不会报证书错误
4. 微信公众平台已配置相同域名为合法 request 域名

#### 当前项目上线前的最后检查

你现在的真实情况是后端已经能通过阿里云 IP 访问，但这还不等于"小程序可上线"。

上线前请再确认：

- 不再使用 `http://47.114.114.135/api` 作为生产地址
- 小程序生产地址已经改成 `https://你的真实域名/api`
- 微信后台已添加相同 HTTPS 域名
- 后端 `.env` 中 `WECHAT_MOCK_MODE=false`
- 后端 `.env` 中已填写 `WECHAT_APP_ID` 和 `WECHAT_APP_SECRET`
- `MEDIA_DRIVER` 不再使用 `local`

## 8. 前后端到底怎么联动

这是最容易搞混的一块，这里说白一点。

### 8.1 管理后台联动流程

例如你在后台新建一个词条：

1. 你在后台页面填表
2. 后台前端发请求到 `/api/admin/dictionary`
3. 后端收到请求
4. 后端把数据写进数据库
5. 下次小程序查词时，就会从数据库读到这个词条

也就是说：

- **后台是写数据的人**
- **数据库是存数据的地方**
- **小程序是读数据的人**

### 8.2 小程序联动流程

例如用户在小程序首页搜索"你好"：

1. 小程序把关键词发给 `/api/miniapp/search`
2. 后端去数据库里查词条、短语、谚语、民歌
3. 后端把结果返回给小程序
4. 小程序把结果显示出来

### 8.3 首页数据联动流程

现在小程序首页不再写死轮播和搜索建议。

而是：

1. 小程序首页请求 `/api/miniapp/home`
2. 后端从数据库里找已发布的民歌封面，生成轮播数据
3. 后端从数据库内容里提取搜索建议词
4. 小程序把这些数据显示出来

所以这块现在已经是：

- **数据库 -> 后端 -> 小程序首页**

## 9. 你应该怎么理解"删除前端硬编码数据"

很多人会误会这句话。

这里真正该删除的，是：

- 本来应该从数据库读取的内容数据
- 例如词条数组、民歌数组、首页轮播内容、搜索建议词池

不一定要删除的，是：

- 页面按钮文字
- 标题文案
- UI 说明语
- 图标路径
- 样式类名

也就是说：

- **业务内容** 应该尽量进数据库
- **界面文案和样式** 不一定需要进数据库

## 11. 最推荐的上线结构

如果你不想折腾，推荐就按这个结构上：

```text
微信小程序
    ↓
https://your-domain.com/api

运营后台浏览器
    ↓
https://your-domain.com/admin-web/

Nginx
    ↓
Node / NestJS
    ↓
MySQL + COS
```

这个结构的好处是：

- 简单
- 稳定
- 不容易跨域
- 后台和接口同域
- 小程序只要配一个 HTTPS 域名就行

## 12. 最后给不会部署的人一句话版本

你只要记住下面这 6 句话：

1. 小程序和后台都不直接连数据库，它们都先连后端。
2. 后端才是真正连接数据库的程序。
3. 后台地址是 `/admin-web/`，接口地址是 `/api/...`。
4. 本地开发可以用 `sqljs`，正式上线一定要换 MySQL。
5. 小程序正式环境一定要用 HTTPS 接口地址。
6. 最省事的部署方法，就是让 Nginx 把同一个域名下的所有请求都转给 NestJS。

## 14. Web 前端怎么部署

项目现在除了小程序和后台管理，还有一个面向用户的 Web 前端（Vue 3 SPA）。

### 14.1 Web 前端和后端的关系

```text
Web 前端 (Vue 3 SPA)
    ↓ Axios HTTP 请求
后端 NestJS (已有)
    ↓
MySQL + COS
```

Web 前端和后台管理前端一样，都是通过 HTTP 请求调用后端接口。
但 Web 前端是一个独立的 SPA 项目，需要单独构建和部署。

### 14.2 本地开发 Web 前端

1. 进入 Web 前端目录：
   ```bash
   cd web
   ```

2. 安装依赖：
   ```bash
   npm install
   ```

3. 启动开发服务器（默认 http://localhost:5173）：
   ```bash
   npm run dev
   ```

4. 开发环境下，Vite 会自动代理 `/api` 请求到 `http://127.0.0.1:3000`，
   所以后端需要先启动（参考第 3 节）。

### 14.3 Web 前端生产构建

```bash
cd web
npm run build
```

构建产物在 `web/dist/` 目录，这是纯静态文件（HTML/CSS/JS）。

### 14.4 Web 前端部署方案

推荐两种方案：

#### 方案 A：Nginx 直接托管静态文件（推荐）

把 `web/dist/` 的内容放到 Nginx 的静态目录：

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /path/to/fullchain.pem;
    ssl_certificate_key /path/to/privkey.pem;

    # Web 前端静态文件
    root /www/buyi-dictionary/web/dist;
    index index.html;

    # SPA 路由回退：所有未匹配的路径返回 index.html
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 请求转发给 NestJS 后端
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 后台管理页面也转发给 NestJS
    location /admin-web/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

这个方案的访问路径：
- 用户访问 `https://your-domain.com` → 看到 Web 前端（布依语词典 Web 版）
- 用户访问 `https://your-domain.com/api/*` → 请求打到 NestJS 后端
- 用户访问 `https://your-domain.com/admin-web/` → 看到后台管理

#### 方案 B：NestJS 后端托管 Web 静态文件

类似后台管理的做法，让 NestJS 也把 Web 前端的构建产物挂出来。

在 backend 的 NestJS 配置中新增一个 ServeStaticModule：

```typescript
ServeStaticModule.forRoot({
    rootPath: join(__dirname, '..', '..', 'web', 'dist'),
    exclude: ['/api/(.*)', '/admin-web/(.*)'],
}),
```

这样只需一个后端服务，同时提供：
- `/` → Web 前端
- `/api/*` → 后端接口
- `/admin-web/` → 后台管理

缺点：每次更新 Web 前端需要重新部署后端。

### 14.5 部署后怎么检查 Web 前端

1. 打开你的域名（如 `https://your-domain.com`）
2. 能看到布依语词典 Web 首页
3. 搜索能查到词条
4. 点击民歌能播放音频
5. 登录/注册功能正常
6. 收藏和学习记录功能正常
