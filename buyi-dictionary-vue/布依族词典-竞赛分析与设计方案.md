## 一、项目概述

### 1.1 项目目标

布依族词典是一个面向布依族语言文化保护的数字媒体应用，提供词汇查询、短语学习、谚语传承、民歌欣赏、文化探索等功能。项目同时包含 Web 端（Vue 3 SPA）和微信小程序端，后端基于 NestJS。

**参赛目标**：2026 年北京市大学生数字媒体设计竞赛 B 类（互联网应用及多媒体作品应用类）。

### 1.2 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| Web 前端 | Vue 3.4 + Vite 5.4 | Composition API + `<script setup>` |
| 状态管理 | Pinia 2.1 | 组合式与选项式混用 |
| 路由 | Vue Router 4.2 | History 模式 + 鉴权守卫 |
| HTTP | Axios 1.6 | 拦截器 + 401 自动刷新 token |
| 样式 | 原生 CSS 变量令牌 | 无 UI 框架，Liquid Glass 设计系统 |
| 小程序前端 | 原生微信小程序 | pages/components/utils 结构 |
| 后端 | NestJS + TypeORM | SQLite(开发) / MySQL(生产) |
| 数据库 | SQLite / MySQL | 开发用 sqljs，生产用 MySQL |

## 三、项目缺陷诊断

### 3.1 表面缺陷（已知问题）

#### 缺陷 1：播放器为模拟实现（严重）

**现状**：[player.js](src/stores/player.js) 使用 `requestAnimationFrame` 模拟进度推进，时长硬编码 222 秒（3:42），未接入真实音频文件。

**影响**：B 类要求"可实际运行的应用程序"，模拟播放会被评委认为功能未完成。后端 `/uploads/buyi_audio/` 目录下已存在真实音频文件（fenshoudiao.MP3、huiyouge.MP3、yushuiqing.MP3 等），但前端未接入。

**严重程度**：致命

#### 缺陷 2：图片资源未压缩（性能）

**现状**：

| 文件 | 大小 | 影响 |
|------|------|------|
| bg-vocabulary.jpg | 1,437 KB | 词典页首屏延迟数秒 |
| bg-profile.png | 916 KB | 个人页首屏延迟 |
| bg-phrases.jpg | 616 KB | 短语页首屏延迟 |
| bouyei-nature.jpg | 365 KB | 文化页偏大 |
| bouyei-textile.jpg | 356 KB | 文化页偏大 |

**影响**：影响页面加载速度和现场演示流畅度。

**严重程度**：高

#### 缺陷 3：内容单薄（竞赛核心扣分点）

**现状**：项目仅提供单词查询、短语、谚语、民歌列表，功能体量不足以支撑"数字媒体"竞赛的丰富度要求。

**影响**：竞赛评审"设计价值"维度考察实用性和创新性，内容单薄直接拉低得分。

**严重程度**：高

#### 缺陷 4：缺少配套说明材料

**现状**：无作品说明书、无演示视频脚本、无项目背景调研文档。

**影响**：竞赛要求提交作品时附带说明材料，辅助评委理解创作初衷。

**严重程度**：中

#### 缺陷 5：Windows 兼容性未验证

**现状**：未在 Windows Chrome/Edge 本地打开构建产物进行测试，SPA 路由回退在本地文件协议下可能失效。

**影响**：竞赛强制要求"Windows 平台可正常运行"。

**严重程度**：高

### 3.2 更大的短板（深层结构性问题）

#### 短板 A：缺乏"数字媒体"的技术创新亮点

**问题**：竞赛是"数字媒体设计竞赛"，不是普通 Web 开发比赛。当前项目本质上是一个"带漂亮 UI 的 CRUD 应用"，缺少数字媒体技术的创新点。

**具体表现**：
- 没有音频可视化（频谱、波形）
- 没有 3D/VR/AR 元素
- 没有AI 驱动的智能功能（仅有 SSE 对话，未突出）
- 没有交互式数据可视化（仅有静态声调曲线图和柱状图）
- 没有动画/动效叙事

**改进方向**：
- 民歌播放增加实时音频频谱可视化（Web Audio API + Canvas）
- 文化探索增加可交互的 3D 纹样旋转查看（CSS 3D 或 Three.js）
- 声调系统增加可弹奏的交互式钢琴键盘
- AI 助手增加语音合成（布依语发音）
- 增加加载序章动画（布依纹样逐步展开的 SVG 路径动画）

#### 短板 B：文化内容停留在"展示"而非"体验"

**问题**：文化探索页面（[Culture.vue](src/views/Culture.vue)）目前是静态图文展示，用户只能被动阅读，无法沉浸式体验布依族文化。

**具体表现**：
- 蜡染、织锦、铜鼓等非遗内容仅有一段文字描述 + 一张图片
- 声调系统有图表但不可交互
- 纹样符号有 SVG 但不可探索
- 没有故事叙事线
- 没有音频/视频多媒体内容

**改进方向**：
- 将文化页从"信息展示"升级为"数字文化馆"
- 增加布依族民俗故事模块（图文 + 朗读音频）
- 增加数字科普板块（服饰、节日、习俗的交互卡片）
- 纹样符号增加点击展开详情（放大图 + 文化溯源 + 历史演变）
- 增加双语对照学习模块

#### 短板 C：工具页与叙事页边界模糊

**问题**：所有页面使用相近的视觉语法，导致产品像统一模板复制品，而非不同功能页有不同气质。

**具体表现**：
- 词典、设置、登录等工具页也有"表演感"
- 首页、文化、民歌等叙事页又不够沉浸
- 玻璃效果全站同权重使用，无主次分层

**改进方向**：
- 叙事页（首页/文化/民歌）：保留图像氛围，增加沉浸感
- 工具页（词典/设置/收藏/记录）：回归清晰可读，弱化玻璃
- 严格执行 hero/content/quiet 三级材质语义

#### 短板 D：动效缺乏编排节奏

**问题**：当前动效"处处都有但处处平庸"，缺少有记忆点的动效设计。

**具体表现**：
- 所有卡片统一 hover 浮起
- 所有玻璃都跟鼠标光斑
- 所有标题都呼吸渐入
- 没有页面切换过渡
- 没有加载序章

**改进方向**：
- 只在关键节点保留动效（首页 Hero 入场、搜索弹出、播放器展开）
- 增加页面切换过渡（View Transitions API 或 CSS 过渡）
- 增加加载序章动画
- 减少非关键区域的动效密度

#### 短板 E：缺少"协同创新"体现

**问题**：竞赛主题是"数字创意，协同创新"，但项目没有体现"协同"维度。

**改进方向**：
- 增加趣味答题排行榜（用户间协同竞争）
- 增加学习社区或分享功能
- 在作品说明书中强调跨学科协作过程

## 五、UI 整体设计方案

### 5.1 风格定位

**核心定位：数字文化馆**

项目不应是"炫光科技站"，而应是"布依族数字文化馆"。视觉语言从"特效感"转向"材质感"，从"每处都在发光"转向"少数地方有重量"。

**设计关键词**：

- 馆藏感：像博物馆展陈，有秩序、有层次、有呼吸
- 民族性：蜡染靛蓝、织锦暖橙、银饰银白，文化符号融入系统
- 克制美：高光面积越小越高级，玻璃越少越显重要
- 信息先行：装饰不能压过搜索、阅读、学习等主任务

**品牌色彩系统**：

| 令牌 | 色值 | 语义 |
|------|------|------|
| `--c-brand` | `#3A6B8C` | 蜡染靛蓝，主品牌色 |
| `--c-accent` | `#D4883A` | 织锦暖橙，强调色 |
| `--c-bg-warm` | 米白系 | 暖背景 |
| `--c-bg-silver` | 银白系 | 冷背景 |
| `--c-text` | `#1B3A5C` | 深靛蓝文字 |
| 深色模式背景 | `#0f1419` | 深色背景 |
| 深色模式品牌色 | `#6BA3BE` | 提亮品牌色 |

**字体系统**：

| 令牌 | 字体 | 用途 |
|------|------|------|
| `--font-serif` | Noto Serif SC | 标题、文化叙事 |
| `--font-sans` | DM Sans | 正文、UI 文案 |
| `--font-mono` | JetBrains Mono | 数据、标签、编号 |

### 5.2 页面架构与性格差异

项目页面分为三类，每类有不同视觉气质：

#### 叙事页（氛围优先）

| 页面 | 定位 | 设计关键词 |
|------|------|-----------|
| 首页 Home | 品牌入口 + 文化馆首屏 | 大图、留白、单一搜索动作、加载序章 |
| 文化 Culture | 内容展陈页 | 图片叙事、纹样交互、声调可弹奏、科普卡片 |
| 民歌 Songs | 沉浸式内容入口 | 封面锚点、频谱可视化、列表清晰 |
| 民俗故事 Stories（新增） | 图文叙事 + 朗读 | 杂志式排版、沉浸阅读、音频嵌入 |

#### 工具页（清晰优先）

| 页面 | 定位 | 设计关键词 |
|------|------|-----------|
| 词典 Dictionary | 阅读型工作页 | 列表扁平、详情聚焦、交互清晰 |

| 答题 Quiz（新增） | 互动答题 | 即时反馈、计分动画、结果展示 |

#### 管理页（轻量优先）

| 页面 | 定位 | 设计关键词 |

| 个人中心 Profile | 信息仪表盘 | 轻玻璃、图表卡、徽章墙 |

### 5.4 材质分层系统

Liquid Glass 已建立三级语义，需严格执行：

| 层级 | 类名 | 用途 | 视觉特征 |
|------|------|------|---------|
| 重点层 | `liquid-glass-hero` | Hero、搜索模态、登录卡 | 更厚、更亮、更有景深 |
| 内容层 | `liquid-glass-content` | 详情面板、学习卡、文化说明 | 轻微模糊，强调可读性 |
| 安静层 | `liquid-glass-quiet` | 统计卡、设置组、列表项 | 几乎不发光，只保留边界 |

**执行原则**：
1. 每一屏只保留一个"最亮"的视觉重点
2. 卡片不要都像玻璃，次要内容宁可更平
3. 高光面积越小，质感越容易高级
4. 玻璃效果越少，越能显出它的重要性

### 5.5 动效策略

从"处处有动效"收敛为"少数关键节点有编排动效"：

**保留的动效**：
- 首页 Hero 入场（staggered reveal，animation-delay 递增）
- 搜索弹窗弹出（scale + opacity 过渡）
- 播放器展开/收起（高度过渡）
- 民歌封面呼吸（playing 状态 4s 循环）
- 滚动入场（IntersectionObserver，threshold 0.15）
- 统计数字计数动画（cubic easing）

**减少的动效**：
- 所有卡片统一 hover 浮起 → 只保留交互卡片的 hover
- 所有玻璃跟鼠标光斑 → 只保留 Hero 区域
- 所有标题呼吸发亮 → 移除
- 多层粒子漂浮 → 精简至 6-8 个

**新增的动效**：
- 加载序章（SVG 路径动画，布依纹样逐步展开）
- 页面切换过渡（View Transitions API 或 CSS fade）
- 答题正确/错误反馈动效（scale bounce + 颜色闪烁）
- 音频频谱可视化（Web Audio API + Canvas real-time）

**动效规范**：
- 所有动效遵循 `prefers-reduced-motion`
- 仅动画 `transform` / `opacity`
- hover 用 `@media (hover: hover)` 限定，避免触屏粘性
- 缓动函数统一使用 `cubic-bezier(0.32, 0.72, 0, 1)`

### 5.6 民族特色表达策略

文化元素不应只停留在"装饰符号"层面，应融入排版秩序、网格结构、插图语言和页面节奏：

| 层级 | 表达方式 | 现状 | 改进 |
|------|---------|------|------|
| 色彩 | 蜡染靛蓝 + 织锦暖橙 + 银饰银白 | ✅ 已有 | 保持 |
| 字体 | Noto Serif SC 用于文化标题 | ✅ 已有 | 保持 |
| 纹样 | PatternDecoration SVG 装饰 | ⚠️ 仅装饰 | 转化为加载动画、分隔线、按钮纹理 |
| 排版 | 展览编号（EXHIBIT 01/02/03） | ✅ 已有 | 扩展到新页面 |
| 内容 | 蜡染/织锦/铜鼓/民歌 | ⚠️ 偏浅 | 增加民俗故事、科普板块、双语对照 |

## 八、总结

### 项目优势

1. **选题优秀**：布依族语言数字化保护，契合"数字创意"主题，有文化价值和社会意义
2. **设计系统成熟**：Liquid Glass + 蜡染靛蓝配色有识别度，三级材质语义已建立
3. **全栈完整**：Vue Web + 微信小程序 + NestJS 后端，覆盖多端
4. **无障碍完善**：ARIA、键盘导航、reduced-motion 支持
5. **架构规范**：Pinia 状态管理、Axios 封装、路由鉴权

### 项目劣势

1. **播放器模拟**：致命问题，不符合 B 类"可实际运行"要求
2. **内容单薄**：仅词典查询，缺少多媒体互动内容
3. **缺少数字媒体创新**：无音频可视化、无 3D 交互、无 AI 亮点
4. **文化体验不足**：静态展示，缺少沉浸式交互
5. **缺少配套材料**：无作品说明书、无演示视频
6. **Windows 兼容性未验证**：竞赛强制要求

### 冲奖策略

以"布依族数字文化馆"为核心概念，通过以下策略冲击一等奖：

1. **修复致命问题**（播放器 + Windows 兼容）
2. **增加数字媒体创新亮点**（音频频谱、交互声调钢琴、加载序章动画）
3. **拓展内容深度**（民俗故事、趣味答题、双语对照、数字科普）
4. **强化民族特色**（纹样融入系统、文化叙事化、音频多媒体化）
5. **完善配套材料**（作品说明书、演示视频脚本）
