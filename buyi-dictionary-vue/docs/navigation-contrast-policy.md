# 导航栏背景对比度 P0 不变量

状态：**锁定（P0）**  
适用范围：桌面端与移动端全局导航栏  
变更权限：只有产品负责人明确提出修改时，才允许同步修改实现、本文档和回归测试。

## 不可更改原则

1. 沉浸式页面滚动后继续保持透明导航栏，不得改成常驻不透明顶栏。
2. 背景判断以导航栏下方的语义区块为事实来源，固定采样视口横向 `14% / 50% / 86%` 三点，至少两票才切换明暗主题。
3. 页面必须用 `data-nav-tone="light|dark"` 标记导航栏可能经过的背景；语义标记必须覆盖对应背景的完整横向区域，允许更具体的嵌套区块覆盖外层基调。
4. 深浅主题切换固定使用 `96ms` 稳定延迟，禁止因单帧或单点结果立即翻转。
5. 探测结果不确定，或标题、正文、链接、按钮、图片等高密度内容经过导航栏下方时，必须保留可靠主题并启用极轻微保护衬底；不得清空主题、隐藏导航或强制猜测新主题。
6. 全局页脚必须声明 `data-nav-tone="dark"`，导航栏接近深色收尾区和页脚时必须使用浅色内容。
7. 导航栏始终固定在视口顶部并保持 `z-index: 40`；滚动状态不得通过 `display`、`visibility`、`opacity` 或移出视口的方式隐藏导航栏。

## 固定工程参数

这些参数集中在 `src/utils/navTonePolicy.js`，由 `tests/navTonePolicy.test.js` 精确校验：

- 三点采样：`[0.14, 0.5, 0.86]`
- 通过票数：`2`
- 切换延迟：`96ms`
- 探测线：导航栏底部向内 `8px`，只判断导航内容实际覆盖的背景，禁止提前切色
- 高密度背景：标题、正文、链接、按钮、图片、视频、画布和 SVG 必须触发保护衬底
- 可靠状态：完全透明
- 不确定状态：保留最后可靠主题并启用轻微模糊衬底

任何违反以上条件的修改都应被视为 P0 回归，测试必须失败，不能带入发布版本。
