## 一、技术架构总览

### 1.1 最终架构

```
┌─────────────────────────────────────────────────────────────────┐
│                         用户访问层                               │
├──────────────────────────┬──────────────────────────────────────┤
│   Web 浏览器              │      微信小程序                       │
│   (评委/用户)             │      (用户手机)                       │
└───────────┬──────────────┴───────────────┬──────────────────────┘
            │                               │
            │ HTTPS                         │ wx.cloud.callFunction
            │ (Cloudflare)                  │ (微信自有通道)
            ▼                               ▼
┌───────────────────┐             ┌──────────────────┐
│ Cloudflare Pages  │             │  云函数 apiProxy │
│ buyi-dictionary   │             │  (微信服务器)    │
│ .pages.dev        │             └────────┬─────────┘
└─────────┬─────────┘                      │
          │                                │ HTTP (IP 直连)
          │ Cloudflare Tunnel              │ 无需备案
          │ (可选)                         │
          └────────────────┬───────────────┘
                           ▼
                 ┌──────────────────────┐
                 │   阿里云服务器         │
                 │   (新服务器待创建)     │
                 ├──────────────────────┤
                 │ Nginx :80/:443       │
                 │ NestJS :3000         │
                 │ MySQL :3306          │
                 └──────────────────────┘
```

### 1.2 核心优势

| 问题 | 解决方案 |
|------|---------|
| 小程序需要备案域名 | **云函数代理**：小程序调云函数，云函数直连后端 IP，绕开白名单限制 |
| Web 端需要 HTTPS | **Cloudflare Pages**：自带可信 HTTPS，无需自签证书 |
| 后端需要稳定运行 | **阿里云 ECS**：国内服务器稳定、低延迟 |
| 成本控制 | 云函数免费额度（4 万次/月）、阿里云试用额度 |

### 1.3 各端访问方式

| 端 | 访问方式 | 是否需要备案 |
|----|---------|------------|
| Web 浏览器 | `https://buyi-dictionary.pages.dev` | ❌ 无需备案 |
| 小程序 | 云函数代理 → 后端 IP | ❌ 无需备案 |
| 后端 API | 云函数直连 `http://<IP>:3000/api` | ❌ 无需备案 |

## 二、后端部署（阿里云服务器）

### 2.1 服务器选型

**推荐：国内阿里云（华东/华北）**

| 配置项 | 推荐值 |
|--------|--------|
| 地域 | 华东 1（杭州）或华北 2（北京） |
| 规格 | 2 核 4G（ecs.c6.large） |
| 系统 | Ubuntu 22.04 64 位 |
| 带宽 | 1-5 Mbps 按流量 |
| 存储 | 40GB 系统盘 |
| 时长 | 试用 1-3 个月（覆盖评审期） |

**注意**：如果选择国内阿里云，Web 端需要走 Cloudflare Tunnel 才能用 HTTPS。如果选择香港/海外，可以直接用 Caddy 签发 Let's Encrypt 证书。

### 2.2 服务器初始化

创建服务器后，SSH 登录：

```bash
ssh root@<服务器公网 IP>
```

执行基础环境安装：

```bash
# 更新系统
apt update && apt upgrade -y

# 安装基础工具
apt install -y git curl wget vim ufw docker.io docker-compose

# 启动 Docker
systemctl enable docker
systemctl start docker

# 配置防火墙
ufw allow 22/tcp
ufw allow 80/tcp
ufw allow 443/tcp
ufw --force enable

# 安装 Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs
npm install -g pm2
```

### 2.3 部署 MySQL（Docker）

```bash
mkdir -p /opt/buyi-dictionary
cd /opt/buyi-dictionary

cat > docker-compose.yml << 'EOF'
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    container_name: buyi-mysql
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: Buyi@Dict2026Root
      MYSQL_DATABASE: buyi_dictionary
      MYSQL_USER: buyi
      MYSQL_PASSWORD: Buyi@Dict2026
    ports:
      - "127.0.0.1:3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
    command: --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci
volumes:
  mysql_data:
EOF

docker-compose up -d
```

### 2.4 上传后端代码

**服务器上执行**：


## 三、Web 前端部署

### 3.1 方案选择

| 方案 | 访问地址 | 优点 | 缺点 |
|------|---------|------|------|
| **Cloudflare Pages** | `https://buyi-dictionary.pages.dev` | 免费 HTTPS、CDN、无需备案 | 首屏约 1 秒延迟 |
| 阿里云 Nginx HTTP | `http://39.96.81.132` | 零延迟 | 浏览器显示"不安全" |

**推荐**：使用 Cloudflare Pages，HTTPS 更适合评委访问。

### 3.2 Cloudflare Pages 部署

**本地执行**：

```powershell
cd d:/BuyiDictionaryWeb/buyi-dictionary-vue

# 配置环境变量（如果需要）
# .env.production 中设置 VITE_API_BASE_URL 指向后端

# 构建
npm run build

# 部署到 Cloudflare Pages
npx wrangler pages deploy dist --project-name=buyi-dictionary
```

部署成功后访问：`https://buyi-dictionary.pages.dev`

### 3.3 配置 API 代理（可选）

如果需要 Web 端也走 Cloudflare Tunnel：

1. 在阿里云服务器安装 cloudflared：

```bash
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64 -o cloudflared
chmod +x cloudflared
mv cloudflared /usr/local/bin/
```

2. 创建 Tunnel：

```bash
cloudflared tunnel login
cloudflared tunnel create buyi-backend
cloudflared tunnel route dns buyi-backend api.buyitech.asia
```

3. 启动 Tunnel：

```bash
cloudflared tunnel run --url http://localhost:3000 buyi-backend
```

## 四、小程序云函数代理配置

### 4.1 核心原理

```
小程序客户端
    │ wx.cloud.callFunction({ name: 'apiProxy', data: { path, method, data } })
    ▼
云函数 apiProxy（运行在微信服务器）
    │ http.request({ hostname: '<阿里云IP>', port: 3000, path, method })
    ▼
阿里云后端
    │
    ▼
返回数据 → 云函数 → 小程序客户端
```

**关键点**：
- 云函数运行在微信服务器，**不受小程序白名单限制**
- 可以直接访问 HTTP IP、未备案域名
- 无需在微信公众平台配置服务器域名

### 4.2 代码已就绪

以下文件已配置完成：

| 文件 | 作用 |
|------|------|
| [`cloudfunctions/apiProxy/index.js`](../BuyiDictionaryApp-main/BuyiDictionaryApp-main/cloudfunctions/apiProxy/index.js) | 云函数代理逻辑，直连阿里云 IP |
| [`utils/runtime-config.js`](../BuyiDictionaryApp-main/BuyiDictionaryApp-main/utils/runtime-config.js) | 云函数配置（envId、functionName） |
| [`app.js`](../BuyiDictionaryApp-main/BuyiDictionaryApp-main/app.js) | requestApi 方法，优先调用云函数 |

### 4.3 配置后端 IP

云函数需要知道阿里云服务器的 IP 地址。修改：

**`cloudfunctions/apiProxy/index.js` 第 9 行**：

```javascript
// 改为你的阿里云服务器公网 IP
const BACKEND_BASE = 'http://<你的阿里云IP>:3000/api';
```

### 4.4 开通云开发

1. 用微信开发者工具打开小程序项目：
   ```
   d:/BuyiDictionaryWeb/BuyiDictionaryApp-main/BuyiDictionaryApp-main
   ```

2. 点击工具栏 **"云开发"** 按钮

3. 点击 **"开通"**，同意协议

4. 创建环境：
   - 环境名称：`buyi-dict`（自定义）
   - 套餐：**基础版（免费）**

5. 记下环境 ID（形如 `cloud1-xxxxxxxx`）

### 4.5 更新环境 ID

修改以下两处：

**1. `utils/runtime-config.js` 第 12 行**：
```javascript
envId: 'cloud1-d3gj9ohe8a7ec0023',  // 改为你的环境 ID
```

**2. `cloudbaserc.json` 第 2 行**：
```json
"envId": "cloud1-d3gj9ohe8a7ec0023"  // 改为你的环境 ID
```

### 4.6 上传云函数

1. 在微信开发者工具左侧文件树，找到 `cloudfunctions` 文件夹

2. **右键** `cloudfunctions` → 选择你的云环境

3. **右键** `cloudfunctions/apiProxy` → **"上传并部署:云端安装依赖"**

4. 等待上传完成

### 4.7 测试云函数

在云开发控制台 → 云函数 → apiProxy → 测试，输入：

```json
{
  "path": "/health",
  "method": "GET"
}
```

预期返回：

```json
{
  "statusCode": 200,
  "data": { "status": "ok", ... }
}
```

## 附录：常见问题

### Q1: 云函数调用失败，提示 FUNCTION_NOT_FOUND？

**原因**：云函数未上传或环境 ID 不匹配。

**解决**：
1. 确认已在微信开发者工具中右键 `cloudfunctions` → 选择正确的云环境
2. 重新上传云函数：右键 `apiProxy` → "上传并部署"
3. 等待 1-2 分钟后再测试

### Q2: 云函数调用成功，但返回 500 错误？

**原因**：后端 IP 配置错误或后端服务未启动。

**解决**：
1. 检查 `cloudfunctions/apiProxy/index.js` 中的 `BACKEND_BASE` 是否为正确的阿里云 IP
2. 在服务器上执行 `pm2 logs buyi-backend` 查看后端日志
3. 测试 `curl http://localhost:3000/api/health`

### Q3: Web 端无法访问后端 API？

**原因**：跨域或 API 地址配置错误。

**解决**：
1. 检查 `.env.production` 中的 `VITE_API_BASE_URL`
2. 确认后端 `CORS_ORIGIN` 配置正确
3. 使用 Cloudflare Tunnel 或改为同域部署

### Q4: 小程序正式版无法调用云函数？

**原因**：正式版环境 ID 与代码中配置不一致。

**解决**：
1. 在微信公众平台查看正式版的云开发环境 ID
2. 更新代码中的 `envId` 并重新上传小程序代码
