# 布依族词典 Web 版 UI 设计评估与优化建议（已落地）

更新时间：2026-07-11

本文档基于当前 Vue 前端实际实现，重新梳理原 `UI设计评估与优化建议.md` 中的判断，明确哪些方向是合理的、哪些优化已经落地、以及后续仍可继续迭代的部分。

---

## 一、先回答核心问题：原文档里的 UI 优化方向是否合理？

结论是：**合理，而且总体上符合项目整体风格与项目现实意义。**

原因有三点：

1. **这个项目不是纯工具站，也不是纯展示页。**
   它同时承担“布依语词典查询”和“文化内容展示”两类任务，所以界面既不能做成冷冰冰的后台，也不适合做成全站炫技型视觉实验。原文档提出“叙事页保留氛围，工具页回归清晰”，这个方向是对的。

2. **项目已经有自己的文化语境。**
   当前项目并不缺品牌感，配色、图片、纹样、民歌内容都已经建立了“数字文化馆”气质。原文档判断问题不在“没有风格”，而在“风格过满、层级过平均”，这个判断与当前代码状态一致。

3. **这个项目真正要服务的是查找、阅读、学习。**
   搜索、结果列表、学习记录、收藏、设置这些功能必须优先保证可读性和操作清晰度。原文档强调“装饰不能压过主任务”，这非常符合项目的现实意义。

因此，这次落地不是推翻旧方向，而是把旧方向从“设计诊断”进一步转成“与代码一致的系统化收尾”。

---

## 二、哪些原始判断被保留？

以下判断在复核真实代码后，仍然成立：

1. **液体玻璃不该全站同权重使用。**
   Hero、详情面板、弹窗、普通列表项不应使用同一强度的材质。

2. **叙事页和工具页应区别对待。**
   首页、文化页、民歌页可以更注重图像氛围；
   词典、学习、收藏、记录、设置等页面要更强调信息扫描和操作效率。

3. **“馆藏感”比“炫技感”更适合这个项目。**
   布依族词典更像数字文化馆，而不是科技演示站。

4. **文化表达不能只停留在装饰层。**
   文化元素应该通过图片、节奏、层级和信息组织进入页面，而不是只靠纹样和发光效果。

## 三、已经落地的设计决策

### 1. 液体玻璃从“单一效果”变成“分层材质”

已在 `src/assets/styles/liquid-glass.css` 中建立三档语义：

1. `liquid-glass-hero`
   用于高关注区域，如登录卡、搜索模态等。

2. `liquid-glass-content`
   用于内容承载面板，如学习信息、详情面板、文化说明区。

3. `liquid-glass-quiet`
   用于统计卡、设置组、列表项等低权重区域。

同时继续收窄了默认玻璃的光学存在感：

- 默认高光更小、更弱
- 噪声层透明度降低
- hover 镜面增强幅度下降
- `glow-card` / `glow-hero` 发光尺寸与透明度继续降低

这一步的目标不是“去掉玻璃”，而是让玻璃有身份，不再满屏平均发声。

### 2. 背景与动效已从“处处活跃”收敛到“少数地方存在”

已落地的收敛包括：

- `src/components/common/PageShell.vue`
  - 默认粒子密度为 `6`
  - 视差系数为 `0.15`
- `src/views/Home.vue`
  - Hero 粒子数降为 `8`
  - 首屏保留图像主导，不再是卡片堆叠
- `src/utils/liquidGlass.js`
  - rect 缓存已存在
  - 滚动镜面偏移进一步收窄
  - 镜面回弹更慢、更稳

这意味着当前页面的氛围感仍在，但信息前景不再被背景特效持续抢走注意力。

### 3. 工具页与叙事页的边界被进一步拉开

#### 叙事页

- `src/views/Home.vue`
  - 保留大图 Hero 与单一搜索入口
  - 文化卡和学习区保持内容层级分化
- `src/views/Culture.vue`
  - 保留真实图片叙事
  - 纹样卡降到更轻的材质层
- `src/views/Songs.vue`
  - 保留首屏封面 + 播放动作的强视觉锚点
  - 文化说明区改为更轻的内容层面板

#### 工具/管理页

- `src/views/Dictionary.vue`
  - 筛选药丸改为安静层
  - 详情面板改为内容层，去掉多余发光
- `src/views/Learn.vue`
  - 学习卡正面升级为重点层
  - 学习统计条和进度条降为轻层级
- `src/views/Settings.vue`
  - 外观设置保留较高权重
  - 其余分组下调为安静层
- `src/views/Favorites.vue`
  - 摘要为内容层
  - 空状态和列表项为安静层
- `src/views/Record.vue`
  - 统计概览与记录项改为安静层
  - 类型分布用内容层承载
- `src/views/Profile.vue`
  - 头像区保留较强层级
  - 统计卡、图表卡、菜单列表整体下调

这样处理后，不同页面终于开始体现真实的职责差异，而不是统一套一层“玻璃滤镜”。

## 五、当前 Web Interface Guidelines 合规状态

基于本轮复核，当前项目在以下方面已明显改善：

### 已符合或基本符合

1. 交互元素语义化
   - 结果项、抽屉按钮、播放按钮、收藏按钮多数已使用正确标签

2. 焦点管理
   - 搜索模态已有焦点陷阱
   - 主要按钮和链接的 `focus-visible` 进一步补齐

3. 减少动效偏好
   - 玻璃、页面 reveal、播放器、首屏动效多数已有 `prefers-reduced-motion` 兜底

4. 图片尺寸与加载策略
   - 多个关键页面图片已有 `width`、`height`、`loading`

5. 文案细节
   - 加载态、占位符等多数已统一使用 `…`

### 仍建议后续继续检查

1. 深色模式下的硬编码 `rgba(...)`
   - 虽然已经减少，但首页、民歌页、部分装饰组件仍有残留

2. 少量装饰性阴影与渐变
   - 某些页面仍可进一步 token 化

3. 图表与复杂组件
   - 当前 `ToneChart.vue` 已具备替代文本，但后续如果继续新增图表，仍要保持同样规范

## 六、各页面当前风格定位

为了避免后续再次回到“全站一个调性”的问题，这里明确当前推荐的页面定位：

1. **首页 `Home.vue`**
   - 定位：品牌入口 + 文化馆式首屏
   - 关键词：大图、留白、单一搜索动作

2. **词典 `Dictionary.vue`**
   - 定位：阅读型工作页
   - 关键词：列表扁平、详情聚焦、交互清晰

3. **学习 `Learn.vue`**
   - 定位：单任务学习界面
   - 关键词：学习卡为中心，统计为辅助

4. **文化 `Culture.vue`**
   - 定位：内容展陈页
   - 关键词：图片叙事、纹样说明、少量辅助材质

5. **民歌 `Songs.vue`**
   - 定位：沉浸式内容入口 + 曲库列表
   - 关键词：封面锚点、列表清晰、文化说明克制

6. **个人中心 / 收藏 / 记录 / 设置**
   - 定位：管理与状态页
   - 关键词：轻玻璃、弱装饰、高可读性

## 七、最终判断

当前 UI 的优化方向没有跑偏。

更准确地说，项目已经从“民族文化主题 + 强特效叠加”的阶段，逐步走向“有文化气质、有系统秩序、主次更清楚”的阶段。

本轮落地后的结果可以概括为：

1. **没有换风格，而是把原风格收得更像产品。**
2. **没有抹掉文化氛围，而是让文化表达回到更合适的位置。**
3. **没有把页面都做平，而是让重点区域真正变得更有重量。**

这比简单地“继续削特效”更重要，因为布依族词典需要的不是失去个性的朴素界面，而是一个既能承载文化内容、又能支持真实查词与学习任务的成熟前端系统。

---

## 八、后续仍建议继续迭代的事项

以下内容不属于本轮强制实施，但值得继续做：

1. 继续清理首页、民歌页、装饰组件中的硬编码 `rgba(...)`
2. 针对暗色模式再做一次逐页视觉走查
3. 对文化类页面引入更稳定的版式节奏，而不只依赖图片与卡片
4. 若后续新增页面，严格复用 `hero / content / quiet` 三级材质，不再回到“新页面先上普通 liquid-glass”的旧习惯
