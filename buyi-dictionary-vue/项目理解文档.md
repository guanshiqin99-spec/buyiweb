## 一、项目目标理解

### 1.1 核心目标

**短期目标（竞赛目标）**：
- 参加「2026 年北京市大学生数字媒体设计竞赛」B 类（互联网应用及多媒体作品应用类）
- 以"布依语言文化数字化保护"为选题，冲击一等奖
- 提交 Windows 可运行的 Web 应用 + 1920×1080 演示视频

**长期目标（推测，待确认）**：
- 保护和传承布依族语言文化
- 为布依族语言学习者、研究者、文化爱好者提供数字化工具
- 可能有实际落地和推广的计划

### 1.2 目标用户（推测，待确认）

- **布依族人民**：日常语言查询、文化传承
- **语言学习者**：学习布依语的学生或爱好者
- **文化研究者**：研究布依族文化的学者
- **竞赛评委**：数字媒体设计竞赛评审专家（当前最重要）
- **普通游客**：对民族文化感兴趣的访客

### 1.3 项目定位

当前定位：**数字媒体竞赛作品**
- 强调视觉冲击力和数字媒体技术创新
- 强调文化价值和社会意义
- 强调交互体验和多媒体内容

## 二、技术框架理解

### 2.1 整体架构

项目采用**前后端分离**架构，包含三个主要部分：

```
┌─────────────────────────────────────────────────┐
│                    用户端                         │
│  ┌──────────────┐    ┌──────────────────┐      │
│  │  Web 端      │    │  微信小程序端     │      │
│  │  (Vue 3 SPA) │    │  (原生小程序)    │      │
│  └──────┬───────┘    └────────┬─────────┘      │
│         │                     │                │
└─────────┼─────────────────────┼────────────────┘
          │                     │
          ▼                     ▼
┌─────────────────────────────────────────────────┐
│                  后端服务                         │
│              (NestJS + TypeORM)                  │
│  ┌──────────┐  ┌──────────┐  ┌──────────────┐ │
│  │ 小程序   │  │ 管理后台 │  │  AI 助手     │ │
│  │ API      │  │ API      │  │ (SSE 流式)   │ │
│  └──────────┘  └──────────┘  └──────────────┘ │
│                   │                             │
└───────────────────┼─────────────────────────────┘
                    │
                    ▼
        ┌──────────────────┐
        │   数据库          │
        │   SQLite / MySQL  │
        └──────────────────┘
```

## 三、UI/UX 设计思路理解

### 3.1 整体设计风格

**核心概念：Liquid Glass + 布依族文化**

将 Apple 风格的液态玻璃（Liquid Glass）设计语言与布依族传统美学（蜡染靛蓝、织锦纹样、银饰质感）融合，打造既有现代科技感又有民族文化底蕴的数字产品。

**设计关键词**：
- 🪟 **液态玻璃**：backdrop-filter 模糊 + 菲涅尔边框高光 + 多层光影
- 🔵 **蜡染靛蓝**：主色调 `#3A6B8C`，源自布依族传统蜡染工艺
- 🧡 **织锦暖橙**：点缀色 `#D4883A`，源自布依族织锦色彩
- ⚪ **银饰银白**：背景色 `#E8E4E1`，呼应银饰文化
- 📜 **馆藏叙事**：以"数字文化馆"为整体叙事框架

### 3.2 设计令牌系统

**色彩令牌**（见 [variables.css](src/assets/styles/variables.css)）：
- 品牌色三级：brand-300 / brand-500 / brand-700
- 强调色：accent-300
- 背景：warm 暖白 / silver 银白
- 文字：深靛蓝为主，多级透明度
- 功能色：success 绿 / danger 红

**字体令牌**：
- `--font-serif`：Noto Serif SC（文化标题、叙事性文字）
- `--font-sans`：DM Sans（正文、UI 文案）
- `--font-mono`：JetBrains Mono（数据、标签、编号）

**材质令牌**（见 [liquid-glass.css](src/assets/styles/liquid-glass.css)）：
- `liquid-glass-hero`：重点层（Hero、搜索模态、登录卡）
- `liquid-glass-content`：内容层（详情面板、学习卡）
- `liquid-glass-quiet`：安静层（统计卡、设置组）

### 3.3 UX 设计原则

1. **无障碍优先**：ARIA 标签、键盘导航、焦点管理、`prefers-reduced-motion`
2. **信息先行**：装饰不压过主任务（搜索、阅读、学习）
3. **动效克制**：只在关键节点有编排动效，避免"处处有动效但处处平庸"
4. **页面性格差异化**：
   - 叙事页（首页/文化/民歌）：氛围优先，沉浸感
   - 工具页（词典/学习）：清晰优先，效率
   - 管理页（个人/收藏/记录/设置）：轻量优先，信息密度

### 3.4 动效设计

**已实现的动效**：
- 首页视差滚动
- 卡片 hover 浮起
- 玻璃元素跟随鼠标光斑
- 搜索弹窗过渡
- 翻转卡 3D 翻转动画
- 民歌封面呼吸动画
- 滚动入场动画（IntersectionObserver）
- 统计数字计数动画

**动效规范**：
- 仅动画 `transform` / `opacity`（GPU 加速）
- 统一缓动：`cubic-bezier(0.32, 0.72, 0, 1)`
- 遵循 `prefers-reduced-motion`
- Hover 仅在 `@media (hover: hover)` 下启用

### 3.5 民族文化表达层次

| 层级 | 表达方式 | 状态 |
|------|---------|------|
| 色彩 | 蜡染靛蓝 + 织锦暖橙 + 银饰银白 | ✅ 已实现 |
| 字体 | Noto Serif SC 衬线体用于文化标题 | ✅ 已实现 |
| 纹样 | PatternDecoration SVG 装饰组件 | ⚠️ 仅装饰性 |
| 排版 | 展览式编号（EXHIBIT 01/02/03） | ✅ 已实现 |
| 内容 | 蜡染/织锦/铜鼓/民歌文化介绍 | ⚠️ 偏静态 |
| 交互 | 声调曲线图、词汇分布图 | ⚠️ 静态图表 |
| 音频 | 民歌播放 | ⚠️ 模拟实现 |

## 五、尚不明确的问题

> 以下是我在理解项目过程中产生的疑问，希望能得到澄清，以便更好地帮助你推进项目。

### 5.1 项目目标相关

1. **除了参加竞赛，项目还有其他长期目标吗？**
   - 比如：是否有实际落地推广的计划？是否有真实用户群体？

2. **目标用户的优先级是什么？**
   - 竞赛评委 > 普通用户？还是说产品本身也希望做好用户体验？

3. **参赛的具体策略是什么？**
   - 更看重"设计效果"还是"设计价值"？
   - 打算用什么作为核心亮点来打动评委？

### 5.2 内容与数据相关

4. **数据量有多大？**
   - 词典里有多少词条？真实的布依语词汇数据来源是什么？
   - 民歌音频有多少首？是真人演唱吗？

5. **文化内容的深度怎么规划？**
   - 目前的文化介绍比较简略，是否有更详细的调研资料可以补充？
   - 民俗故事、服饰介绍等新增内容，有参考资料吗？

6. **AI 助手的具体定位是什么？**
   - 目前的 SSE 对话是调用什么模型？回答质量如何？
   - 打算怎么突出 AI 作为数字媒体创新亮点？

### 5.3 设计方向相关

7. **为什么选择 Liquid Glass 风格？**
   - 是一开始就确定的方向，还是尝试过其他风格？
   - 有没有考虑过更"民族风"的设计（比如更强调蜡染纹理、传统纹样）？

8. **首页的重构方向你认同吗？**
   - "数字文化馆"的定位是否符合你的预期？
   - 加载序章动画、减少区块数量这些调整你觉得合理吗？

9. **动效的尺度偏好？**
   - 是希望更克制（博物馆式的安静感），还是更炫酷（科技感）？

### 5.4 技术与实现相关

10. **后端和前端的联调状态如何？**
    - Web 端的接口都对接好了吗？还是说很多是 mock 数据？
    - 音频文件在后端有哪些？怎么命名的？

11. **小程序和 Web 端的关系？**
    - 哪个是重点？竞赛提交主要展示哪个？
    - 两者功能需要保持一致吗？

12. **部署环境是怎样的？**
    - 竞赛演示是本地运行还是有线上服务器？
    - 是否需要我帮忙部署到服务器？

### 5.5 团队与流程相关

13. **团队情况是怎样的？**
    - 几个人的团队？各自负责什么？
    - 你在其中是什么角色？

14. **目前的开发节奏如何？**
    - 距离 9 月 20 日截止还有两个多月，计划怎么安排？
    - 哪些是必须做的，哪些是可选的？

15. **我在这个项目中扮演什么角色？**
    - 你希望我主要做什么？写代码？做设计？写文档？还是全方位协助？

1. 民俗故事模块
